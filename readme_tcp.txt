=============================================
              Task1: TCP Socket
=============================================

运行环境：
- Python 3.8+
- 无需额外依赖（仅使用标准库 socket、struct、threading）

=============================================
              文件列表
=============================================

- tcp_server.py    : TCP服务端程序
- tcp_client.py    : TCP客户端程序
- test.txt         : 测试文本文件（需提前准备）

=============================================
              运行步骤
=============================================

1. 启动TCP服务端：
   python tcp_server.py

   默认监听端口：12345
   监听地址：0.0.0.0（所有网络接口）
   支持多客户端并发连接（使用线程）

2. 启动TCP客户端：
   python tcp_client.py <server_ip> <server_port> <Lmin> <Lmax> [seed]

   参数说明：
   - server_ip   : 服务端IP地址（如 127.0.0.1）
   - server_port : 服务端端口（如 12345）
   - Lmin        : 每个数据块的最小长度（正整数，>0）
   - Lmax        : 每个数据块的最大长度（正整数，>= Lmin）
   - seed        : （可选）随机种子，用于复现相同的分块结果

   示例：
   python tcp_client.py 127.0.0.1 12345 50 100
   python tcp_client.py 127.0.0.1 12345 50 100 42   # 指定随机种子

=============================================
              功能说明
=============================================

服务端（tcp_server.py）：
- 创建TCP Socket监听指定端口
- 接收客户端连接，为每个客户端创建独立线程处理
- 接收Initialization报文，获取数据块总数
- 回复Agree报文确认连接
- 接收reverseRequest报文，对数据进行反转处理
- 回复reverseAnswer报文返回反转后的数据
- 支持多客户端并发连接

客户端（tcp_client.py）：
- 读取test.txt文件内容
- 将文件内容随机分割为若干数据块（块长度在[Lmin, Lmax]范围内）
- 与服务端建立TCP连接
- 发送Initialization报文告知数据块总数
- 等待服务端Agree确认
- 逐块发送数据请求反转
- 接收反转结果并打印
- 生成完整反转文件 reversed_output.txt（将服务端返回的反转数据块按顺序拼接）

=============================================
              报文格式
=============================================

所有报文采用固定8字节头部 + 可变长度负载：

头部格式（8字节，网络字节序）：
  - Type (4字节, unsigned int) : 报文类型
  - Length (4字节, unsigned int) : 负载数据长度

报文类型：
  - Type=1: Initialization（客户端→服务端）
           负载: 4字节整数，表示数据块总数
  - Type=2: agree（服务端→客户端）
           负载: 无（Length=0）
  - Type=3: reverseRequest（客户端→服务端）
           负载: 待反转的文本数据（ASCII编码）
  - Type=4: reverseAnswer（服务端→客户端）
           负载: 反转后的文本数据（ASCII编码）

=============================================
              输出文件
=============================================

- reversed_output.txt : 由客户端生成的完整反转文件
  生成规则：将每个数据块经服务端反转后，按原始顺序拼接得到
  输出位置：与程序同目录

- run_log_tcp_server.txt : 服务端运行日志
- run_log_tcp_client.txt : 客户端运行日志
  记录客户端和服务端的详细通信过程，包含时间戳

=============================================
              注意事项
=============================================

1. 运行前确保 test.txt 文件存在于程序目录下
2. 客户端和服务端的端口号需保持一致
3. 服务端使用 0.0.0.0 监听所有网络接口
4. 数据块长度参数 Lmin 和 Lmax 需满足 0 < Lmin <= Lmax
5. 建议使用 Wireshark 抓取网络报文进行分析
